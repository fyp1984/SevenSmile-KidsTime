# 七笑果-宝贝食光-邀请成员接口文档-V1.0

## 1. 适用范围

- 适用模块：`家庭成员 / 邀请新成员 / 权限配置 / 身份验证绑定`
- 适用角色：`家长管理员`
- 当前实现形态：高保真原型内置 `Mock API`，用于演示真实后端接口契约与交互闭环

## 2. 目标能力

- 支持新增 `父母 / 祖辈 / 阿姨 / 康复人员 / 宝贝` 五类成员
- 支持基础档案、关系信息、联系信息、健康档案、照料需求采集
- 支持邀请链接、二维码、验证码生成与发送
- 支持邀请接受、身份验证、成员绑定、权限分级
- 支持成员 CRUD、邀请状态追踪、过期与撤回
- 支持敏感字段加密存储与掩码展示

## 3. 鉴权要求

- 所有接口默认需要主账号家长身份
- 请求头建议：

```http
Authorization: Bearer <family-admin-token>
X-Role: parent
X-Family-Id: family_001
```

- 未通过鉴权时返回：

```json
{
  "code": 403,
  "message": "仅家长可执行成员管理"
}
```

## 5. 接口清单

### 5.1 查询成员列表

- 方法：`GET /api/family/members`
- 用途：按类型、角色、状态筛选家庭成员

查询参数：

```json
{
  "type": "grand",
  "role": "执行者",
  "status": "已加入"
}
```

返回示例：

```json
{
  "code": 0,
  "message": "ok",
  "data": [
    {
      "id": "member-001",
      "name": "外婆",
      "type": "grand",
      "role": "执行者",
      "status": "已加入"
    }
  ]
}
```

### 5.2 新增成员档案

- 方法：`POST /api/family/members`
- 用途：直接录入成员档案

请求体：

```json
{
  "type": "aunt",
  "name": "李阿姨",
  "relation": "住家阿姨",
  "ownerRelation": "日间照护",
  "phone": "13812345678",
  "wechat": "ayi-lee",
  "email": "",
  "idNo": "330102199001011234",
  "healthSummary": "需了解坚果过敏",
  "careNeeds": "负责午餐备餐与午睡后加餐",
  "rehabProgress": "",
  "role": "执行者",
  "notifyVia": "仅微信"
}
```

说明：

- `phone / wechat / email / idNo` 入库前必须加密
- `rehabProgress` 仅康复人员强制要求

### 5.3 更新成员档案

- 方法：`PATCH /api/family/members/:id`
- 用途：编辑基础信息、联系信息、健康档案、照料说明

### 5.4 删除成员

- 方法：`DELETE /api/family/members/:id`
- 用途：移除非宝贝成员

说明：

- 宝贝成员删除需走多宝贝档案流程，不在本轮原型删除范围内

### 5.5 生成邀请

- 方法：`POST /api/family/invitations`
- 用途：生成带时效的邀请链接、二维码、验证码

请求体：

```json
{
  "type": "rehab",
  "name": "陈康复师",
  "relation": "康复师",
  "ownerRelation": "康复支持",
  "phone": "13987654321",
  "wechat": "rehab-chen",
  "email": "rehab@example.com",
  "idNo": "310101198808088888",
  "healthSummary": "主要跟进步态训练与核心控制",
  "careNeeds": "每周同步一次训练反馈",
  "rehabProgress": "第4周完成80%",
  "role": "协作者",
  "notifyVia": "微信 + 站内",
  "channels": ["邮箱", "微信"],
  "validityHours": 168,
  "verificationMode": "邮箱验证"
}
```

返回示例：

```json
{
  "code": 0,
  "message": "created",
  "data": {
    "id": "invite-002",
    "status": "待接受",
    "link": "https://demo.7smile.cn/member-invite/rehab002",
    "qrText": "7SMILE://invite/rehab002",
    "verifyCode": "582604"
  }
}
```

### 5.6 发送邀请

- 方法：`POST /api/family/invitations/send`
- 用途：通过短信、微信、邮件发送邀请

请求体：

```json
{
  "inviteId": "invite-002",
  "channels": ["邮箱", "微信"]
}
```

### 5.7 邀请接受与身份验证

- 方法：`POST /api/family/invitations/accept`
- 用途：被邀请人点击链接后完成身份验证并绑定主账号

请求体：

```json
{
  "inviteId": "invite-002",
  "verifyCode": "582604"
}
```

返回示例：

```json
{
  "code": 0,
  "message": "accepted",
  "data": {
    "inviteStatus": "已接受",
    "memberId": "member-009"
  }
}
```

说明：

- 若成员类型为 `child`，接受后还需同步写入多宝贝档案
- 若邀请码已过期，返回 `410`

### 5.8 权限配置

- 方法：`PATCH /api/family/permissions`
- 用途：更新成员权限等级

请求体：

```json
{
  "memberId": "member-009",
  "role": "协作者"
}
```

## 6. 敏感信息处理规范

- 加密字段：
  - `phone`
  - `wechat`
  - `email`
  - `idNo`
  - 健康档案中的敏感备注
- 展示策略：
  - 列表页只显示 `mask`
  - 编辑页按权限解密回填
  - 邀请页只显示时效、渠道、验证码，不回显完整明文

## 7. 权限分级

- `管理员`：成员管理、规则配置、设备管理、愿望审核、健康编辑
- `协作者`：照护回填、健康编辑、愿望审核
- `执行者`：查看方案、照护执行、提醒接收
- `只读`：仅查看餐单、教程、提醒
- `宝贝`：成长视角、愿望、签到、乐园

## 8. 失败场景

- 联系方式全部为空：返回 `400`
- 康复人员未填康复进度：返回 `400`
- 非家长调用：返回 `403`
- 邀请不存在：返回 `404`
- 邀请已过期：返回 `410`
- 验证码错误：返回 `422`

## 9. 当前原型映射

- 成员中心页面：`家庭成员`
- 邀请表单：全屏抽屉 `member-invite`
- 身份验证：全屏抽屉 `member-verify`
- 模拟接口日志：成员中心底部 `接口审计`
